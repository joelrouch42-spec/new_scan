1. 🎯 VUE D'ENSEMBLE

Objectif: Scanner automatiquement les marchés financiers, détecter des signaux de trading basés sur des indicateurs techniques, et exécuter des ordres via Interactive Brokers (IBKR).

Fonctionnalités Principales

  • 🔄 Scanning temps réel de 100+ symboles (NASDAQ 100, crypto)
  • 📈 Indicateur technique: Squeeze Momentum (LazyBear)
  • 🎯 Signaux couleur basés sur changements de momentum
  • 🛡️ Stop-Loss intelligent avec gestion différenciée ouverture/continu
  • 📊 Trading automatisé via IBKR avec gestion des positions
  • 📁 Tracking complet des positions et niveaux SL
  • 📉 Génération de graphiques pour visualisation des signaux

4. 🌅 PREMIER SCAN DE LA JOURNÉE

Déclenchement: Premier lancement du jour OU scan_status.json.date != aujourd'hui

Séquence Complète

ÉTAPE 1: DÉTECTION NOUVELLE JOURNÉE ├── Lecture scan_status.json ├── Si scan_status.date < "2025-12-04": │ ├── 🗓️ Print "Nouvelle journée détectée" │ ├── cleanup_data() → Supprime data/ et chart/ │ ├── makedirs("data/") → Recrée dossiers │ └── Mise à jour scan_status.json: │ { │ "date": "2025-12-04", │ "opening_scan_done": true │ } └── return TRUE → is_opening_scan = true
ÉTAPE 2: TÉLÉCHARGEMENT DONNÉES (pour chaque symbole) ├── Cache vide (cleanup effectué) ├── filename = "data/2025-12-04_AAPL_200_1d.csv" ├── df = download_ibkr_data("AAPL", 200, "1d") └── df.to_csv(filename) → Sauvegarde cache
ÉTAPE 3: ANALYSE TECHNIQUE ├── Calcul Squeeze Momentum │ ├── Bollinger Bands (20, 2.0) │ ├── Keltner Channel (20, 1.5) │ ├── Momentum = linreg(source_diff, 20) │ └── Couleurs: lime, green, red, maroon │ ├── Détection signaux couleur │ ├── BUY: momentum devient MAROON (increasing negative) │ ├── SELL: momentum devient GREEN (decreasing positive) │ └── Index du signal sur DataFrame │ └── Filtrage bougie cible ├── target_idx = len(df) - 1 - test_candle └── Garde seulement signaux sur cette bougie
ÉTAPE 4: GESTION STOP-LOSS OUVERTURE ├── Connexion IBKR unique ├── is_opening_scan = TRUE ├── Prix ouverture = df.iloc[-1]['Open'] ├── current_prices = {symbol: {'Open': prix}} │ ├── Vérification SL touchés (première fois = vide) │ ├── sl_hits = check_sl_hits_opening(current_prices) │ └── return [] car position_tracking.json vide/inexistant │ └── Pas de SL à traiter au premier scan
ÉTAPE 5: EXÉCUTION SIGNAUX ├── Signal détecté (ex: SELL AAPL) ├── Print "🔴 SELL AAPL @ $279.34 - 2025-12-04" │ ├── Tentative trading │ ├── trader.connect() → "✅ Connecté" │ ├── trade_result = smart_trade("AAPL", "SELL") │ │ ├── can_trade("AAPL", "SELL") → Check position size │ │ ├── position_size = 0 (aucune position) │ │ ├── if shares <= 0: return None │ │ └── Print "❌ Aucune position à vendre" │ └── trader.disconnect() → "🔌 Déconnecté" │ ├── NOUVEAU: Calcul et sauvegarde SL AVANT smart_trade() ├── handle_new_signal() → Sauvegarde dans position_tracking.json ├── trade_result = smart_trade() (peut échouer mais SL déjà sauvé) └── ✅ SL maintenant loggé même si ordre échoue

5. 🔄 SCANS CONTINUS

Déclenchement: Tous les scans après le premier scan de la journée

Différences avec le Premier Scan

SCAN CONTINU (is_opening_scan = FALSE) ├── Pas de cleanup ├── Utilisation cache existant OU nouveaux téléchargements ├── scan_status.json déjà marqué opening_scan_done = true │ ├── Gestion SL différente: │ ├── Prix temps réel = df.iloc[-1]['Close'] │ ├── current_prices = {symbol: {'Current': prix}} │ ├── check_sl_hits_continuous(current_prices) │ │ ├── SL + tolérance 1% │ │ └── Fermeture immédiate si touché │ └── Pas de vérification nouveaux signaux │ └── Nouveaux signaux traités normalement

Protection Crash (Scan Continu)

// Logique SL avec tolérance if(action == "BUY") { sl_with_tolerance = sl_level * (1 - tolerance/100); // 1% plus bas sl_hit = current_price <= sl_with_tolerance; } else { // SELL sl_with_tolerance = sl_level * (1 + tolerance/100); // 1% plus haut sl_hit = current_price >= sl_with_tolerance; } if(sl_hit) { // Fermeture immédiate sans vérification signaux opposite_action = (action == "BUY") ? "SELL" : "BUY"; smart_trade(symbol, opposite_action); remove_position_tracking(symbol); print("🚨 CRASH PROTECTION: position fermée"); }

6. 🛡️ GESTION STOP-LOSS

Deux Modes de Fonctionnement

Aspect Scan Ouverture Scan Continu
Déclenchement Premier scan du jour Scans suivants
Prix Utilisé Prix d'ouverture (Open) Prix temps réel (Close)
SL Testé Niveau exact (sans tolérance) Niveau + tolérance 1%
Si SL Touché Vérifier nouveaux signaux → Garder ou fermer Fermeture immédiate
Objectif Stratégie intelligente Protection crash

Calcul des Niveaux SL

// Calcul SL selon direction function calculate_sl_level(action, high_signal, low_signal, high_previous, low_previous) { if(action == "BUY") { // Position longue: SL = max des highs return max(high_signal, high_previous); } else { // SELL // Position courte: SL = min des lows return min(low_signal, low_previous); } } // Exemple concret: // Signal BUY sur bougie: High=180.50, Low=178.20 // Bougie précédente: High=182.00, Low=179.10 // SL = max(180.50, 182.00) = 182.00

Stratégie Intelligente (Scan Ouverture)

LOGIQUE AMZN OCTOBRE: ├── Position BUY ouverte le 01/10 avec SL = 180.00 ├── 02/10: Prix ouverture = 174.50 (SL touché!) │ ├── Vérifier: Nouveau signal BUY aujourd'hui ? → OUI │ ├── Action: Garder position + Mettre à jour SL = 176.50 │ └── Print "🔄 Nouveau signal BUY - SL mis à jour" │ ├── 03/10: Prix ouverture = 172.00 (SL touché!) │ ├── Vérifier: Nouveau signal BUY aujourd'hui ? → OUI │ ├── Action: Garder position + Mettre à jour SL = 174.00 │ └── Print "🔄 Nouveau signal BUY - SL mis à jour" │ └── 04/10: Prix ouverture = 170.50 (SL touché!) ├── Vérifier: Nouveau signal BUY aujourd'hui ? → NON ├── Action: Fermer position (SELL) └── Print "🚨 SL OUVERTURE déclenché: position fermée"

7. 📊 DÉTECTION DES SIGNAUX

Indicateur Squeeze Momentum (LazyBear)

Port exact du code Pine Script SQZMOM_LB de TradingView

Paramètres de l'Indicateur

Paramètre Valeur Description
bb_length 20 Période Bollinger Bands
bb_mult 2.0 Multiplicateur Bollinger Bands
kc_length 20 Période Keltner Channel
kc_mult 1.5 Multiplicateur Keltner Channel
use_true_range true Utiliser True Range pour KC

Calcul du Momentum

// Calcul source pour momentum highest_high = highest(high, kc_length); lowest_low = lowest(low, kc_length); sma_close = sma(close, kc_length); avg1 = (highest_high + lowest_low) / 2; avg2 = (avg1 + sma_close) / 2; source_diff = close - avg2; momentum = linreg(source_diff, kc_length);

Attribution des Couleurs

function get_momentum_color(momentum_values) { colors = []; prev_val = 0; for(val in momentum_values) { if(val > 0) { if(val > prev_val) { colors.append("lime"); // Increasing positive } else { colors.append("green"); // Decreasing positive ← SELL } } else { if(val < prev_val) { colors.append("red"); // Decreasing negative } else { colors.append("maroon"); // Increasing negative ← BUY } } prev_val = val; } return colors; }

Génération des Signaux

function detect_signals(momentum, colors) { signals = []; for(i = 1; i < colors.length; i++) { prev_color = colors[i-1]; curr_color = colors[i]; // BUY SIGNAL: Momentum devient MAROON if(curr_color == "maroon" && prev_color != "maroon") { signals.append({ index: i, type: "bullish", momentum: momentum[i], color: curr_color }); } // SELL SIGNAL: Momentum devient GREEN if(curr_color == "green" && prev_color != "green") { signals.append({ index: i, type: "bearish", momentum: momentum[i], color: curr_color }); } } return signals; }
